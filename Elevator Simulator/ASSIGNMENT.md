<p><strong>Course:</strong>&nbsp;Intermediate C++ Programming&nbsp;<strong>University Inspiration:</strong>&nbsp;University of Waterloo&nbsp;<strong>Project Type:</strong>&nbsp;Team Project (up to 3 students)</p>
<p><strong>Project Overview:</strong></p>
<p>In this project, your team will design and implement a simulation of an elevator system using C++. The simulation should model the behavior of a multi-floor building with one or more elevators, handling user requests, managing elevator movement, and simulating real-time actions. The main goal is to practice object-oriented programming (OOP), system design, and interactive console input/output.</p>
<p><strong>Learning Objectives:</strong></p>
<ul>
<li>Apply object-oriented principles (classes, inheritance, polymorphism).</li>
<li>Practice modular design and system decomposition.</li>
<li>Implement time-based or event-driven simulations.</li>
<li>Strengthen skills in C++ syntax, control flow, and basic file I/O.</li>
</ul>
<p><strong>Team Guidelines:</strong></p>
<ul>
<li>Teams may consist of&nbsp;<strong>1 to 3 students</strong>.</li>
<li>Every team must submit one project per group.</li>
<li>Roles should be divided to ensure each student contributes.</li>
</ul>
<p><strong>Functional Requirements:</strong></p>
<ol>
<li><strong>Building Configuration:</strong></li>
<ul>
<li>Configurable number of floors (e.g., 5 to 20).</li>
<li>One or more elevators.</li>
</ul>
<li><strong>Elevator Behavior:</strong></li>
<ul>
<li>Moves up/down between floors.</li>
<li>Responds to user requests (floor buttons and external up/down calls).</li>
<li>Opens/closes doors.</li>
<li>Can be idle if no requests.</li>
</ul>
<li><strong>Request System:</strong></li>
<ul>
<li>Simulate people pressing buttons on different floors.</li>
<li>Handle multiple requests efficiently.</li>
</ul>
<li><strong>Console Interface:</strong></li>
<ul>
<li>Display current state of elevators (floor, direction, status).</li>
<li>Allow user to input requests during runtime.</li>
</ul>
<li><strong>Timing/Simulation:</strong></li>
<ul>
<li>Time-based or step-based simulation.</li>
<li>Delays between actions for realism.</li>
</ul>
</ol>
<p><strong>Optional Enhancements (Bonus):</strong></p>
<ul>
<li>Prioritize requests based on proximity or direction.</li>
<li>Add capacity limits to elevators.</li>
<li>Implement a basic GUI using a library like SFML or ncurses (optional).</li>
<li>Add elevator sound or floor announcements.</li>
</ul>
<p><strong>Deliverables:</strong></p>
<ul>
<li>Source code in C++ (with comments).</li>
<li>A README file explaining:</li>
<ul>
<li>How to compile and run the simulation</li>
<li>Team members and their roles</li>
<li>Features implemented</li>
</ul>
<li>A short demo video (max 3 minutes) showing the simulation in action.</li>
</ul>
